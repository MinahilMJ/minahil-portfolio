A simplified Candy Crush-style game developed in Assembly Language to explore low-level programming, keyboard input handling, game logic, and memory management.

Assembly Language Game Development Keyboard Interrupts Low-Level Programming

01 / OVERVIEW

This project involved developing a simplified Candy Crush-style game using Assembly Language.

The project focused on implementing core game mechanics while working directly with low-level programming concepts such as memory, registers, keyboard input, and program execution.

02 / KEY FEATURES

  • Tile-based game board.
  • Keyboard-controlled player interaction.
  • Keyboard interrupt handling for input.
  • Tile matching logic.
  • Score calculation.
  • Board updates after game actions.
  • Core game state management.

03 / LOW-LEVEL CONCEPTS

  • Keyboard interrupt handling.
  • Register manipulation.
  • Memory management.
  • Low-level input processing.
  • Program flow and control logic.
  • Debugging Assembly Language programs.

04 / TECHNOLOGIES

Language

Assembly Language

Input

Keyboard Interrupts

Application Type

Game

Programming Level

Low-Level Programming

05 / PROJECT OUTCOME

The project strengthened my understanding of computer architecture and low-level programming by requiring direct interaction with registers, memory, keyboard input, and program execution.

It also provided practical experience in implementing game logic without relying on high-level programming abstractions.
